## Running Benchmarks and Generating Plots for HTUVM, HTOVS and CUCO

1. Run the benchmark scripts located in the `benchmark_scripts` directory.
   - Separate subdirectories are provided for **HTUVM** and **HTOVS**.
   - Within each subdirectory, individual scripts are available for each benchmark kernel:
     - `insert` — used to generate **Figure 9 (left-hand side)**.
     - `delete` — used to generate **Figure 9 (right-hand side)**.
     - `insert_sort` — used to generate **Figure 12**.
     - `search_positive` — used to generate **Figure 10 (left-hand side)** and **Figure 15**.
     - `search_negative` — used to generate **Figure 10 (right-hand side)**.

2. After all benchmark runs have completed, execute the `hashtable_fig9_fig10_fig12_fig15_plot.py` script located in the `Graph_Plotting_Scripts` directory:

   ```bash
   python3 hashtable_fig9_fig10_fig12_fig15_plot.py \
       <results_HTOVS/results_HTUVM/results_CUCO> \
       <results_HTOVS/results_HTUVM/results_CUCO> \
       <insert/delete/insert_sort/search_positive/search_negative> \
       <insert/delete/insert_sort/search_positive/search_negative> \
       <insert/delete/search>
   ```

### Arguments

| Argument | Description |
|----------|-------------|
| **1** | Path to the first results directory (`results_HTOVS`, `results_HTUVM`, or `results_CUCO`) |
| **2** | Path to the second results directory (`results_HTOVS`, `results_HTUVM`, or `results_CUCO`) |
| **3** | Benchmark type of first directory: `insert`, `delete`, `insert_sort`, `search_positive`, or `search_negative` |
| **4** | Benchmark type of second directory: `insert`, `delete`, `insert_sort`, `search_positive`, or `search_negative` |
| **5** | Kernel type: `insert`, `delete`, or `search` |

> **Note**
>
> - The script generates comparison plots for the selected benchmark configuration.
> - Although the benchmark type may differ (e.g., `search_positive` vs. `search_negative`), the kernel comparison remains the same.
> - Using the **same results directory** with different benchmark types is only required for reproducing **Figure 12**.

## GPH Configuring Oversubscription


For details about the GPH design and evaluation, refer to:

> Jiaping Cao, Le Xu, Man Lung Yiu, Jianbin Qin, and Bo Tang. 2025. 
> GPH: An Efficient and Effective Perfect Hashing Scheme for GPU Architectures. 
> Proceedings of the ACM on Management of Data (PACMMOD), 3(3), Article 165, 26 pages. 
> https://doi.org/10.1145/3725406

To simulate different oversubscription scenarios for GPH, update the value on **line 72** of `geph_exp.cuh`:

`gph_opensource_copy_original/perf/geph_exp.cuh`

| Value | Oversubscription |
|------:|:-----------------|
| 12.8  | 25%              |
| 13.33 | 50%              |
| 13.72 | 75%              |
| 14    | 100%             |

## Build and Run GPH

Follow these steps to build and run GPH:

1. Open a terminal in the GPH root directory.

2. Generate the build files:
   ```bash
   cmake -B build -DCMAKE_POLICY_VERSION_MINIMUM=3.5
   ```

3. Move to the `common` directory:
   ```bash
   cd common
   ```

4. Run CMake:
   ```bash
   cmake
   ```

5. Return to the GPH root directory:
   ```bash
   cd ..
   ```

6. Build the project:
   ```bash
   cmake --build build
   ```

7. Move to the performance benchmark directory:
   ```bash
   cd build/perf
   ```

8. Run the executable:
   ```bash
   ./perf
   ```

> **Note**
>
> Before running `./perf`, make sure to:
> - Generate the traces by running the trace generation scripts.
> - Run `generate_pos_neg_workload.py` in the `data_source` directory.

> **Important**
>
> If you update the trace file path or change the memory reservation, you must rerun the following commands:
>
> ```bash
> cd ../../
> cmake --build build
> cd build/perf
> ./perf
> ```


## Generating Figure 14

To compare HTOVS against GPH, HTOVS must be run with the same configurations used for GPH.

1. Uncomment lines **130–140** in the file:

   ```
   gpu/hashtable/driver_hetero_hash_batch.cu
   ```

2. Update the value on **line 132** according to the oversubscription configuration table provided for GPH.

The execution output will provide the insert kernel execution time in the following format:

```
Total time taken by insert kernel (ms): <value>
```

Use the generated `<value>` from the execution output and manually update the insert timings for both GPH and HTOVS.

In the file:

```
Graph_Plotting_Scripts/hashtable_fig14_plot_manual.py
```

Update the values at the following locations:

- **GPH**: Lines **21, 22, and 23**
- **HTOVS**: Lines **29, 30, and 31**

After updating the values, run the plotting script to generate **Figure 14**.
